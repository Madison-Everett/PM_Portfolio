Dominosa Constraint Satisfaction Project - Dec 2018

Technologies/Languages used:

  • Python
  • Pysearch

Project Description:

This was a project that consisted of building an Artificial Intelligence (AI) model with the goal of being able to solve the puzzle Dominosa, a Constraint-Satisfaction toy problem. In this project, I used various algorithms, such as Backtracking Search, Forward-Checking, Conflict-Directed Backjumping, in order to create a successful and efficient problem-solving model. After a larger amount of testing, I came to the conclusion that the Backtracking Search algorithm with added heuristics performed more optimally and efficiently than a human player.

Machine-Learning Settlement/Failure project at UBS (United Bank of Switzerland) - June 2019

Technologies/Languages used:

  • Python
  • Pandas
  • Matplotlib
  • Tensorflow
  • NumPy

Project Description:

This project was developed during my 10 week internship at UBS. The goal of this project was to create a machine-learning model that could predict whether or not a financial trade would settle or fail. In order to create training data, I had access to one of UBS’s many trading databases that contained CID (Client Identifying Data) (which is one of the reasons why I do not have access to the repo 🙁) and several attributes that could contribute to whether or not a trade was going to settle or fail.

In order to target which trades would fail, I first gathered info from the various subject matter experts around the office in order to pinpoint exactly where trades could go wrong. Secondly, I created a correlation matrix using Pandas and Matplotlib in order to see the correlation of attributes that lead to a failure. From there, I developed my machine learning model into a classification model using the Tensorflow Library and numPy and created my training and test data. This took about 4 weeks. On the first model run, the model performed at 40% accuracy, which was pretty low to start, but all models start somewhere. With longer training sessions, by the end of the 10 weeks, the model performed at 85% accuracy, which was an excellent improvement.

SVD-NMF - Topic Extraction - Jan 2020

Technologies/Languages used:

  • Python
  • Scikit-learn
  • Sklearn
  • Jupyter Notebook
  • MATLAB

Project Description:

This project was an exploration in Text Mining comparing matrix decomposition methods, Singular Value Decomposition (SVD) and Nonnegative Matrix Factorization (NMF) to see which method performed better in summarizing a large collection of texts.

In order to perform this exploration, I used the Scikit-learn and Sklearn framework that contain dummy sets of text data as the corpus of text for the methods. In order to evaluate and compare these methods, I gathered the general topic extraction results, in addition to the results of the two above methods and displayed them in Word Clouds using MATLAB. As a result, it seemed that the NMF method performed similarily to the general topic extraction than SVD, which means that the NMF performs more closely to a human than SVD.

Python Data Migration- for VERA using SQL and APIs Feb 2021-Oct 2021

Technologies/Languages used:

  • Python
  • SQL
  • Xlwt
  • Xlsxwriter
  • Base64
  • Requests
  • Json.decoder

Project Description:

These scripts are from the Data Migration phase of the VERA project. This phase consisted of pulling historical and current data from the current Grants and Agreement system and loading it into the new system using a SFTP server. My task was to pull all PDF files from the Agreement system in order to have them ready for the new system. I completed this task by creating SQL to pull the file names and unique file ids of the files that we needed.

This step had to be done first since the only way to pull the pdfs was through a custom built API that required a “get” containing the document id from the db and the bearer token. From there, I decoded the files using json and base64 and saved the pdfs in a folder system (“For Final copies___" scripts) with the file names that I pulled from the db to prepare them for the new Agreements system. In addition to this, I also had to update a db table in order to keep the files organized for the import into the new system (“filepath___" scripts). I performed this task over 8 months in batches since this process needed to be refined for go live in Oct 2021 and the PDF data was needed for UAT, which began in February 2021.
